# Category 10: Workplace & Productivity

Templates for converting hand-drawn sketches into professional visuals, UI prototypes, magazine layouts, presentation slides, and business documents.

---

## 10.1 Hand-drawn Sketch → Corporate Flowchart

**When to use:** User has a whiteboard sketch or rough diagram and wants it transformed into a professional presentation-ready chart.

```
Convert this hand-drawn whiteboard sketch into a professional corporate flowchart suitable for a [McKinsey / BCG / executive board] business presentation.

Style Guide: Use a minimalist "McKinsey-style" aesthetic — clean lines, ample whitespace, and a sophisticated [blue-and-gray / green-and-gray / monochrome] color palette.

Structure: Automatically align all boxes and diamonds to a strict grid. Connect them with straight, orthogonal arrows (90-degree angles only, no curvy lines). Decision diamonds should be clearly differentiated from process boxes.

Text: Transcribe the handwritten labels into a clear, bold [Arial / Roboto / Inter] sans-serif font. Correct any obvious spelling from context.

Output: High-resolution vector-style image on a pure white background. Suitable for direct drop into a PowerPoint or PDF.
```

## 10.2 Wireframe Sketch → High-Fidelity UI Mockup

**When to use:** User has a rough app or website wireframe and wants a realistic UI design mockup.

```
Transform this rough wireframe sketch into a high-fidelity UI design mockup for a [mobile app / web dashboard / tablet interface].

Design System: Apply a modern, clean aesthetic similar to [iOS 18 / Material Design 3 / Fluent Design]. Use rounded corners, soft drop shadows, and a [vibrant primary color — e.g., cobalt blue / emerald green / coral red].

Component Intelligence: Interpret the sketch intelligently:
- Turn scribbles into high-quality [placeholder images / icon sets]
- Convert rough rectangles into proper buttons with [gradients / flat fills / outlined styles]
- Transform lines into realistic typography blocks with appropriate hierarchy

Layout: Ensure perfect padding (8px grid system) and consistent spacing between all elements.

Context: Place the final design inside a realistic [iPhone 16 / Samsung Galaxy S25 / MacBook Pro 16"] device frame mockup. Add a subtle [drop shadow / soft environment reflection] around the device.
```

## 10.4 Data → Financial Sankey Diagram

**When to use:** User wants to visualize financial flows, income statements, or data pipelines as a Sankey/flow diagram.

```
[Subject]: A professional financial Sankey diagram visualizing the [Income Statement / Revenue Breakdown / Budget Flow / Data Pipeline] of [COMPANY / ORGANIZATION / PROJECT], in the style of "App Economy Insights" and US corporate financial reports.

[Visual Style]: High-fidelity vector infographic, clean minimalist aesthetic, flat design. Background: clean, very light grey or off-white.

[Color Strategy — CRITICAL]:
Analyze the [BRAND NAME] logo. Extract its primary brand color. Use this as the dominant theme for main revenue flows and profit blocks. Create a harmonious palette: saturated shades for vertical nodes (bars), semi-transparent lighter gradients for flowing paths. Avoid clashing colors.

[Composition & Structure]:
- Flow direction: horizontal, Left (Revenue Sources) → Right (Net Profit / Output)
- Path texture: "silky smooth" Bezier curves — liquid ribbons, not jagged lines
- Iconography (left side): minimalist flat vector icons representing each business segment
- Branding: place the official [BRAND] logo clearly at top center

[Data to visualize]:
[INSERT REVENUE/EXPENSE DATA HERE OR DESCRIBE THE FLOW]

[Details]: High resolution, 4K, sharp sans-serif typography, professional data visualization. Clearly distinguish between Revenue, Gross Profit, Operating Expenses, and Net Profit.
```

## 10.6 Photo → Translation / Sign Conversion

**When to use:** User has a photo with text in a foreign language and wants it translated in-place.

```
Translate the [Chinese / Japanese / Arabic / Spanish / Korean] [menu / sign / poster / label / instructions] in the uploaded photo into [English / target language] for [tourists / customers / general readers].

Texture Preservation — CRITICAL: Maintain the original [aged, greasy, painted, printed, handwritten] look of the surface. The new [English / target] text should look like it was [written / printed / painted] on the same surface, with slight [fading / wear / texture] to match. Do not replace the surface — only replace the text characters.

Formatting rules:
- Keep all [prices / numbers / symbols] exactly as they are
- Align translated text to match the original layout and line breaks
- Match [font weight / approximate style — bold, italic, handwritten] of the original
- Do not change image composition, colors, or non-text elements
```
